# docuseal-go

Go SDK for the DocuSeal API. The entire SDK is generated by Fern
(fern-go-sdk): flat client with context-first methods and functional options
(`option.WithAPIKey`, `WithBaseURL`, `WithMaxAttempts`), `<OperationId>Params`
request objects, retries built in. Nothing is written by hand - this
replaced an oapi-codegen types + hand-client hybrid.

## Architecture

- Repo root is the generated `docuseal` package (types.go, environments.go,
  pointer.go helpers like `docuseal.String`/`docuseal.Int`) plus generated
  `client/`, `option/`, `core/`, `internal/` packages. Never edit by hand;
  regenerate with `./generate-types.sh [spec-path-or-url]` (Node.js, Docker
  and ruby required; the Fern generator runs locally in Docker, no account).
- `fern/` - Fern workspace config: generator version pin,
  `packageName: docuseal`, module path.
- `generate-types.sh` preprocesses the spec with ruby (same transform as
  docuseal-java/dotnet): drops webhook schemas, swaps legacy
  `POST /submissions` for `/submissions/init` (envelope response), strips
  tags for a flat surface, injects `x-fern-request-name` for
  `<OperationId>Params` naming. It also strips Fern's bundled tests
  (wiremock/, *_test.go) and meta docs, then runs `go mod tidy` so testify
  does not leak into go.mod.
- Models: required fields are plain values, optional are pointers; each
  struct carries a private explicitFields bitmask + Set* methods (Fern's
  explicit-null mechanism) - literal construction works fine.

## Source of truth

The OpenAPI spec is generated by `lib/api_spec/` in the docuseal-website
repository and served at https://console.docuseal.com/openapi.yml. Do not
fix type issues here - fix them in the spec and regenerate.

## Conventions

- Client surface is flat: `c.GetTemplate(ctx, id)`,
  `c.CreateSubmission(ctx, params)`; method names from spec operationIds.
- Build: `go build ./...`. No tests yet.
