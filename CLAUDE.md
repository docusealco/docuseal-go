# docuseal-go

Go SDK for the DocuSeal API. Hybrid architecture: models are generated from
the OpenAPI spec, the client is written by hand.

## Architecture

- `models/` - generated by openapi-generator (models only), never edit by
  hand. Regenerate with `./scripts/generate-types.sh [spec-path-or-url]`.
  Model naming follows the spec: schemas extracted into OpenAPI components
  get domain names (`TemplateField`, `TemplateAuthor`, ...), inline schemas
  get operation-derived names (`GetSubmissionResponse`,
  `CreateSubmissionResponseInner`, ...). As components are rolled out per
  endpoint in the spec, names improve on regeneration.
- `client.go` - hand-written core: `Client`, options (base URL for
  Global/EU/on-premises), `do()` with auth header and error handling,
  `queryValues()` params serialization.
- `api.go` - hand-written API methods, one per endpoint. Mirrors the method
  naming of docuseal-js (`listTemplates` -> `ListTemplates`).
- `params.go` - hand-written query params structs (a client concern, not in
  models output) and the response type of `POST /submissions/init`, which is
  not in the public spec.

## Source of truth

The OpenAPI spec is generated by `lib/api_spec/` in the docuseal-website
repository and served at https://console.docuseal.com/openapi.yml. Shared
component schemas live in `lib/api_spec/schemas.rb`. Do not fix type issues
here - fix them in the spec and regenerate.

## Conventions

- `CreateSubmission` posts to `/submissions/init` (newer response shape),
  matching docuseal-js. The legacy `POST /submissions` array response is not
  exposed.
- `PermanentlyDelete*` methods use the DELETE endpoints with
  `?permanently=true`.
- Identical inline schemas are merged by openapi-generator under one name
  picked from an arbitrary endpoint (e.g. `ArchiveSubmission` returns
  `ArchiveTemplateResponse`). Extracting those schemas into named
  components in the spec fixes the naming.
- Generated models validate `required` properties on JSON unmarshal, so the
  spec's `required` lists must match actual API responses.
- `scripts/generate-types.sh` patches one openapi-generator bug: invalid Go
  emitted for `default: false` on the oneOf `mask` field preference.
- `--inline-schema-name-mappings` in the script drops HTTP status codes
  from synthesized names (`getTemplate_200_response` -> `GetTemplateResponse`).
  When adding an endpoint, extend the list: new inline names are printed as
  "Inline schema created as ..." warnings during generation.
